子系统 A：WorldBook Studio（世界书工作台）

导入优先的设定工作流。你可以直接粘贴或上传完整世界书，后台多 Agent（解析员、规则员、一致性审查员）会协同解析并输出可运行的结构化设定；仅在缺项时才进行补丁补全。

1. 导入世界书（JSON / YAML / Markdown）

输入一段世界书片段进行模拟解析。正式产品中可直接上传完整世界书文件并进行结构化校验。

2. 多 Agent 解析与一致性审查

解析员 Agent 提取结构字段，规则员 Agent 归纳硬规则，审查员 Agent 检测冲突并输出补丁建议。

核心底层支撑与系统演进

作为一站式全栈应用，CanonWeave 的落地依赖于标准化协议、长期记忆机制以及严谨量化测试，确保单人酒馆中的多 Agent 行为可解释、可追溯。

🔌 MCP 工具链与 ReAct 框架

Agent 不再是“空想家”。遵循严格的 Thought -> Action -> Observation 链条。通过标准化 MCP (模型上下文协议) 操作实体数据。

  • ▪
    MCP-World-Reader/Writer: 原生对接用户导入的世界书。精准查询词条，并在解析与补丁完成后自动格式化写入本地系统。
  • ▪
    MCP-Dice-Roller: 提供真实 D20/D100 数值检定，基于环境状态返回判定结果，支撑硬核规则结算。

🧠 全局共享记忆系统 (Shared Memory)

突破单次 Session 限制，构建向量数据库与图数据库结合的混合检索记忆系统。实现真正的“连续世界体验”。

记忆分层架构：
  • 工作记忆 (LangGraph State): 当前场景上下文，立即响应。
  • 情节记忆 (Vector DB): 历史事件总结，通过相似度召回。
  • 语义记忆 (知识图谱): 动态更新的世界人物关系网，支持逻辑推理。
